# Satang Infrastructure

The `infrastructure` directory contains Terraform code for provisioning core AWS resources for the service.

## Current Architecture

Terraform currently provisions:

- One DynamoDB table with composite primary key (`PK`, `SK`)
- Three global secondary indexes (`GSI1`, `GSI2`, `GSI3`)
- One shared IAM role for Lambda execution
- Multiple Lambda functions (operation-specific modules)
- One API Gateway HTTP API with multiple routes integrated to Lambda

## Directory Layout

```text
infrastructure/
├── README.md
└── terraform/
		├── main.tf                 # AWS provider and environment-aware endpoint config
		├── variables.tf            # Input variables
		├── dynamodb.tf             # DynamoDB table and GSIs
		├── iam.tf                  # Shared Lambda IAM role and DynamoDB permissions
		├── lambda_module.tf        # Lambda module wiring
		├── api_gateway.tf          # HTTP API, routes module wiring, and default stage
		├── outputs.tf              # Root outputs
		├── terraform.tfvars        # Environment variable values
		├── lambda/
		│   └── <operation>/
		│       ├── lambda_<operation>.tf
		│       ├── variables.tf
		│       └── outputs.tf
		├── api_route/
		│   └── <operation>/
		│       ├── main.tf
		│       └── variables.tf
		└── terraform.tfstate.d/    # Local workspace state directories
```

Notes:

- `terraform.tfstate.d/` is generated by Terraform when using workspaces.
- Lambda zip artifacts are expected at `aws/lambda/<operation>.zip`.

## Resource Details

### Provider and Environment Modes

`main.tf` supports two deployment modes via `environment`:

- `dev` (or any non-`local` value): deploys to AWS using supplied credentials.
- `local`: targets local service endpoints (for local AWS emulators) and uses test credentials.

Local endpoints are controlled by `local_endpoints` in `variables.tf`.

### DynamoDB

Resource: `aws_dynamodb_table.dynamodb_table`

- Billing mode: `PROVISIONED`
- Read/Write capacity: `2/2`
- Primary key: `PK` + `SK`
- GSIs:
	- `GSI1` on `GSI_PK` + `GSI_SK`
	- `GSI2` on `GSI2_PK` + `GSI2_SK`
	- `GSI3` on `GSI3_PK` + `GSI3_SK`

All GSIs use `projection_type = "ALL"` with provisioned capacity `2/2`.

### IAM

`iam.tf` provisions:

- A shared Lambda execution role
- `AWSLambdaBasicExecutionRole` managed policy attachment
- Inline DynamoDB access policy with:
	- `PutItem`, `GetItem`, `UpdateItem`, `DeleteItem`, `Query`
	- Access to table ARN and all index ARNs

### Lambda and API Routing Pattern

- Each Lambda is defined in its own module under `terraform/lambda/<operation>/`.
- Each API route integration is defined in `terraform/api_route/<operation>/`.
- `api_gateway.tf` creates one HTTP API (`$default` auto-deploy stage) and wires route modules to Lambda module outputs.
- Each route module creates:
	- API integration (`AWS_PROXY`, payload format `2.0`)
	- Route (`<METHOD> <PATH>`)
	- Lambda invoke permission for API Gateway

## Variables

Defined in `terraform/variables.tf`:

- `region` (default: `ap-southeast-1`)
- `table_name` (default: `satang-dynamodb`)
- `environment` (default: `local`)
- `local_endpoints` (default local endpoint map)
- `access_key` (sensitive, default empty)
- `secret_key` (sensitive, default empty)

Current `terraform.tfvars` sets:

```hcl
region      = "ap-southeast-1"
table_name  = "satang-dynamodb"
environment = "dev"
```

## Prerequisites

- Terraform `1.5+`
- AWS credentials with permissions for API Gateway, Lambda, IAM, and DynamoDB
- Go toolchain (for building Lambda artifacts)
- `zip` command available in your shell

## Build Lambda Artifacts

From repository root:

```bash
make zip-lambdas
```

This builds all Lambda entry points under `app/lambda/**/main.go` and generates zip artifacts in `aws/lambda/`.

## Deploy

From `infrastructure/terraform`:

```bash
terraform init
terraform workspace select dev || terraform workspace new dev
terraform fmt
terraform validate
terraform plan -var-file=terraform.tfvars
terraform apply -var-file=terraform.tfvars
```

For local emulation, set `environment = "local"` (for example in `terraform.tfvars`) and ensure local endpoints are running.

## Destroy

```bash
terraform destroy -var-file=terraform.tfvars
```
